表结构详情

users

用户表 - 存储所有注册用户的基本信息
字段名 类型 约束 默认值 说明
id INT PK AUTO_INCREMENT - 用户唯一标识，自增主键
username VARCHAR(50) NOT NULL UNIQUE - 登录用户名，不可重复
password VARCHAR(255) NOT NULL - 登录密码（建议 BCrypt 加密存储）
nickname VARCHAR(50) - NULL 用户昵称/显示名称，可修改
avatar VARCHAR(255) - /default-avatar.png 头像文件路径或 URL
created_at TIMESTAMP - CURRENT_TIMESTAMP 账号注册时间，自动填充
last_online TIMESTAMP - NULL 最后在线时间，用于显示在线状态

friends

好友关系表 - 存储用户之间的好友关系
字段名 类型 约束 默认值 说明
id INT PK AUTO_INCREMENT - 关系记录唯一标识
user_id INT FK NOT NULL - 发起方用户 ID，关联 users.id
friend_id INT FK NOT NULL - 目标方用户 ID，关联 users.id
created_at TIMESTAMP - CURRENT_TIMESTAMP 好友关系建立时间
注意：当前好友关系是单向记录。如果 A 添加 B 为好友，只插入一条 (A, B) 记录。查询双方共同好友时需要同时匹配 user_id 和 friend_id 两个方向。如需双向关系，可考虑添加触发器自动插入 (B, A) 记录。

messages

消息表 - 存储所有私聊消息记录
字段名 类型 约束 默认值 说明
id INT PK AUTO_INCREMENT - 消息唯一标识
sender_id INT FK NOT NULL - 发送者用户 ID，关联 users.id
receiver_id INT FK NOT NULL - 接收者用户 ID，关联 users.id
content TEXT NOT NULL - 消息内容（文本/图片URL/文件路径等）
is_read TINYINT(1) - 0 是否已读：0=未读, 1=已读
created_at TIMESTAMP - CURRENT_TIMESTAMP 消息发送时间
扩展建议：当前 content 为 TEXT 类型，仅支持纯文本。如需支持图片、语音、文件等多媒体消息，建议增加 msg_type 字段（如 text/image/file/voice），content 统一存 JSON 或对应资源路径。

user_groups

群组表 - 存储聊天群的基本信息
字段名 类型 约束 默认值 说明
id INT PK AUTO_INCREMENT - 群组唯一标识
name VARCHAR(100) NOT NULL - 群名称
owner_id INT NOT NULL - 群主用户 ID（未建立外键，建议添加）
created_at TIMESTAMP - CURRENT_TIMESTAMP 群组创建时间

group_members

群成员表 - 存储群组与成员的多对多关系
字段名 类型 约束 默认值 说明
group_id INT PK NOT NULL - 群组 ID（联合主键之一）
user_id INT PK NOT NULL - 成员用户 ID（联合主键之一）
joined_at TIMESTAMP - CURRENT_TIMESTAMP 加入群组的时间

外键关系

1 : N users.id ← friends.user_id 用户可以拥有多个好友关系
1 : N users.id ← friends.friend_id 用户可以被多个用户添加为好友
1 : N users.id ← messages.sender_id 用户可以发送多条消息
1 : N users.id ← messages.receiver_id 用户可以接收多条消息
M : N user_groups.id ↔ users.id 通过 group_members 表关联，一个群有多个成员，一个用户可加入多个群

常用 SQL 查询

用户注册
INSERT INTO users (username, password, nickname) VALUES ('zhangsan', '$2a$10$hashedPassword...', '张三');
用户登录验证
SELECT id, username, nickname, avatar FROM users WHERE username = 'zhangsan' AND password = '$2a$10$hashedPassword...';
添加好友
-- 添加双向好友关系 INSERT INTO friends (user_id, friend_id) VALUES (1, 2); INSERT INTO friends (user_id, friend_id) VALUES (2, 1);
查询好友列表
SELECT u.id, u.username, u.nickname, u.avatar, u.last_online FROM friends f JOIN users u ON f.friend_id = u.id WHERE f.user_id = 1 ORDER BY u.last_online DESC;
发送消息
INSERT INTO messages (sender_id, receiver_id, content) VALUES (1, 2, '你好，在吗？');
查询与某人的聊天记录（分页）
SELECT m.id, m.content, m.is_read, m.created_at, u.nickname AS sender_name, u.avatar AS sender_avatar FROM messages m JOIN users u ON m.sender_id = u.id WHERE (m.sender_id = 1 AND m.receiver_id = 2) OR (m.sender_id = 2 AND m.receiver_id = 1) ORDER BY m.created_at DESC LIMIT 20 OFFSET 0;
查询未读消息数（按发送者分组）
SELECT sender_id, COUNT(*) AS unread_count FROM messages WHERE receiver_id = 1 AND is_read = 0 GROUP BY sender_id;
标记消息已读
UPDATE messages SET is_read = 1 WHERE sender_id = 2 AND receiver_id = 1 AND is_read = 0;
创建群组并添加成员
-- 1. 创建群 INSERT INTO user_groups (name, owner_id) VALUES ('开发小组', 1); SET @group_id = LAST_INSERT_ID(); -- 2. 添加群成员 INSERT INTO group_members (group_id, user_id) VALUES (@group_id, 1), (@group_id, 2), (@group_id, 3);
查询群成员列表
SELECT u.id, u.username, u.nickname, u.avatar, gm.joined_at FROM group_members gm JOIN users u ON gm.user_id = u.id WHERE gm.group_id = 1 ORDER BY gm.joined_at ASC;
查询我加入的群组
SELECT g.id, g.name, g.created_at, (SELECT COUNT(*) FROM group_members WHERE group_id = g.id) AS member_count FROM user_groups g JOIN group_members gm ON g.id = gm.group_id WHERE gm.user_id = 1;

优化建议

1 群聊消息表 - 当前 messages 只支持私聊。建议增加 group_messages 表或在 messages 中添加 group_id 字段以支持群聊。
2 消息类型 - 建议增加 msg_type 字段（ENUM: text/image/file/voice/video），content 存储对应类型的数据。
3 好友状态 - 建议在 friends 表增加 status 字段（0=待确认, 1=已接受, 2=已拒绝），实现好友申请/验证流程。
4 索引优化 - 为 messages 表的 (sender_id, receiver_id, created_at) 添加联合索引，提升聊天记录查询性能。
5 外键约束 - user_groups.owner_id 和 group_members 的 group_id/user_id 建议添加外键约束，保证数据一致性。
6 会话表 - 建议增加 conversations 表，记录每个会话的最后一条消息、未读数等，避免每次都聚合查询 messages 表。
